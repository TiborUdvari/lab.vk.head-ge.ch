05 - Y-axis with adaptive extent

Same filter pattern as 02 - Data joins, now with a y-axis on the left. As the mode changes, the filtered balls render and the axis re-scales to fit them.

Build-up:

  1. SVG with the margin convention so axis labels have somewhere to live.
  2. Scales derived from filteredBalls — every change re-computes them.
  3. The y-axis is its own cell, idempotent via .data([null]).join("g"), transitioning when the scale changes.
  4. Circles + labels do the data join with enter/update/exit.

Dataset

let balls = [
  {name: "Ping Pong",  mm: 40},
  {name: "Golf",       mm: 43},
  {name: "Squash",     mm: 40},
  {name: "Billiard",   mm: 57},
  {name: "Tennis",     mm: 67},
  {name: "Cricket",    mm: 72},
  {name: "Baseball",   mm: 73},
  {name: "Lacrosse",   mm: 63},
  {name: "Softball",   mm: 97},
  {name: "Volleyball", mm: 210},
  {name: "Soccer",     mm: 220},
  {name: "Basketball", mm: 240},
  {name: "Bowling",    mm: 217}
];

SVG with margin convention

margin.left = 60 makes room for the y-axis tick labels. margin.bottom = 50 makes room for the ball-name labels under each circle.

const w = 900;
const h = 400;
const margin = {top: 30, right: 30, bottom: 50, left: 60};
const innerW = w - margin.left - margin.right;
const innerH = h - margin.top - margin.bottom;

const svg = d3.create("svg")
  .attr("width", w)
  .attr("height", h)
  .attr("viewBox", `0 0 ${w} ${h}`)
  .style("background-color", "white");

const g = svg.append("g")
  .attr("transform", `translate(${margin.left}, ${margin.top})`);

Try it

Pick a mode. Watch the y-axis ticks re-scale to match what’s visible.

const mode = view(Inputs.radio(["all", "small", "big"], {value: "all", label: "Show"}));
display(svg.node());

Filter — same pattern as in 02

const filteredBalls = balls.filter(b => {
  if (mode === "all") return true;
  if (mode === "small") return b.mm < 100;
  if (mode === "big") return b.mm >= 100;
});

Scales — derived from filteredBalls

Because yScale depends on filteredBalls, it re-computes whenever mode changes. That’s how the axis “adapts”: the scale changes, the axis cell sees a new yScale, the axis re-renders.

.nice() rounds the domain to clean tick values (e.g. [0, 97] → [0, 100]).

const cx = (d, i) => (i + 0.5) * innerW / filteredBalls.length;

// each ball gets one horizontal cell — minus a small gap so they don't touch
const cellWidth = innerW / filteredBalls.length;
const gap = 6;
const maxDiameter = Math.max(0, cellWidth - gap);

// constrain the y-axis range so the biggest ball's pixel diameter
// can't exceed one cell. yTop = innerH - maxDiameter (clamped ≥ 0).
// When ball count or max-mm changes, the y-axis "shrinks" automatically.
const yTop = Math.max(0, innerH - maxDiameter);

const yScale = d3.scaleLinear()
  .domain([0, d3.max(filteredBalls, d => d.mm) ?? 1])
  .range([innerH, yTop])
  .nice();

// radius in pixels derived from the same yScale that drives the axis,
// so the top of each circle sits exactly on the gridline = its diameter
const radius = d => (innerH - yScale(d.mm)) / 2;

Y-axis

Idempotent join on a single &lt;g class="y-axis"&gt; so re-runs don’t stack new axes. The transition on .call(d3.axisLeft(yScale)) interpolates tick positions between the old and new scale.

{
  const axisG = g.selectAll("g.y-axis")
    .data([null])
    .join("g")
      .attr("class", "y-axis");

  // adapt tick count to the available y-pixels — ~30 px per tick
  const [bottom, top] = yScale.range();
  const tickCount = Math.max(2, Math.floor((bottom - top) / 30));

  // tickSize(-innerW) extends each tick across the chart width = gridlines
  axisG.transition().duration(750)
    .call(d3.axisLeft(yScale)
      .ticks(tickCount)
      .tickSize(-innerW)
      .tickPadding(8));

  // soften the gridline color, hide the vertical axis spine
  axisG.selectAll(".tick line")
    .attr("stroke", "#bbb")
    .attr("stroke-dasharray", "2 3");
  axisG.select(".domain").remove();
}

Circles + labels

Standard data join keyed by d.name. Enter circles grow from r=0 at the bottom, exit circles shrink and remove. Update circles transition to their new (cx, cy).

{
  // circles — bottom-aligned, radius derived from yScale so the top of
  // each circle sits on the gridline equal to its diameter
  g.selectAll("circle.ball")
    .data(filteredBalls, d => d.name)
    .join(
      enter => enter.append("circle")
        .attr("class", "ball")
        .style("fill", "tomato")
        .style("fill-opacity", 0.6)
        .attr("cx", cx)
        .attr("cy", innerH)
        .attr("r", 0)
        .attr("opacity", 0),
      update => update,
      exit => exit.transition().duration(500)
        .attr("opacity", 0)
        .attr("r", 0)
        .remove()
    )
    .transition().duration(750)
      .attr("cx", cx)
      .attr("cy", d => innerH - radius(d))   // bottom on baseline
      .attr("r", radius)                     // diameter = ball's mm in scale units
      .attr("opacity", 1);

  // labels under each circle
  g.selectAll("text.label")
    .data(filteredBalls, d => d.name)
    .join(
      enter => enter.append("text")
        .attr("class", "label")
        .text(d => d.name)
        .attr("text-anchor", "middle")
        .attr("dominant-baseline", "hanging")
        .attr("font-size", 11)
        .style("fill", "black")
        .attr("x", cx)
        .attr("y", innerH + 8)
        .attr("opacity", 0),
      update => update,
      exit => exit.transition().duration(500)
        .attr("opacity", 0)
        .remove()
    )
    .transition().duration(750)
      .attr("x", cx)
      .attr("y", innerH + 8)
      .attr("opacity", 1);
}

How the pieces fit

  • Cell 6 creates the SVG and inner g (margin convention) — runs once.
  • Cell 8 is the radio input → mode, the single source of state.
  • Cell 9 displays the live SVG node — placed right after the input so the controls and the chart are visually adjacent.
  • Cell 11 derives filteredBalls from balls + mode.
  • Cell 13 derives yScale from filteredBalls. Both 11 and 13 re-run on every mode change.
  • Cell 15 redraws the y-axis. Depends on yScale. Idempotent — same &lt;g&gt; reused, transition interpolates tick movement.
  • Cell 17 redraws circles + labels. Depends on filteredBalls and yScale. Standard data join with three-arg .join().

Adding a second axis (e.g. an x-axis with ball names rotated) would be one more cell that depends on a band-scale derived from filteredBalls.map(d => d.name). Same pattern, no edits to existing cells.
